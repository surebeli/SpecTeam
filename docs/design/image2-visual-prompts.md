# Image2 Visual Prompts

These prompts are for regenerating the core marketing visuals with a SpecTeam-first brand while preserving Phoenix as the internal codename theme.

## Brand Direction

- Market-facing product name: SpecTeam
- Codename motif: Phoenix
- Core message: AI-native spec review and decision alignment
- Tone: precise, high-trust, technical, warm, not gamer, not generic AI neon
- Visual language: dark navy environment, ember-orange energy, teal data accents, premium editorial typography, crisp product UI

## Prompt 1 - Square Logo Hero

Create a square 1:1 brand hero for SpecTeam.

The product name must be "SPECTEAM". Do not use "Phoenix Git" or "AI collaboration tool" anywhere.

Use Phoenix only as a codename motif: a refined phoenix-inspired emblem that also suggests documents, branching decisions, and alignment nodes. The mark should feel modern, abstract, and product-grade rather than mascot-like. Pair it with the text lockup:

- small overline: "CODENAME PHOENIX"
- primary wordmark: "SPECTEAM"
- subtitle: "AI-native spec review and decision alignment"

Composition:

- dark deep-navy background with subtle technical texture
- emblem on the left, typography on the right
- soft ember glow around the emblem, faint blue halo behind the wordmark
- premium launch-art quality, crisp edges, minimal clutter

Palette:

- background: navy, midnight blue, graphite
- primary accent: ember orange to red gradient
- secondary accent: teal-cyan data accent
- text: warm off-white

Negative constraints:

- no literal phoenix mascot illustration
- no "Phoenix Git" text
- no generic AI brain, circuit-board wallpaper, or sci-fi chrome
- no purple-heavy palette
- no low-contrast foggy typography

## Prompt 2 - Square Product Mockup Hero

Create a cinematic square 1:1 product mockup for SpecTeam, shown on a premium desktop monitor in a dark studio environment.

This is a Git-native control surface for AI-native spec review and decision alignment. The UI should communicate divergence review, proposed resolutions, and finalized decisions.

The interface must read as SpecTeam, not Phoenix Git.

Required UI copy:

- top-left brand: "SPECTEAM"
- small badge: "CODENAME PHOENIX"
- hero heading: "ALIGN SPECS. SHIP DECISIONS."

Suggested UI structure:

- top navigation with workflow, review, docs, enterprise
- status chips for open divergences, proposed alignment, resolved decisions
- main card for divergence review with one active issue
- side or secondary card for alignment thread / decision ledger
- clear CTA such as "Review Divergences" or "Finalize Decision"

Design goals:

- premium SaaS launch visual
- dark interface with ember-orange highlights and teal data accents
- subtle phoenix geometry in the background, but do not turn it into a gaming aesthetic
- readable typography, believable layout, product photography quality
- sharper and more executive than a hackathon landing page

Negative constraints:

- no "Phoenix Git" labels
- no zero-code slogan
- no fake code-generation dashboard as the main story
- no excessive lens flare, no plastic 3D icons, no stock-crypto aesthetic

## Prompt 3 - Variation for Website Hero

Create a wide website hero for SpecTeam with the same brand direction as above.

The scene should emphasize:

- multi-agent product and engineering collaboration
- divergence detection across PRD, API, and architecture decisions
- clear progression from open issue to aligned decision

The interface should feel like a command center for shared decision quality, not a code editor clone.
