# Test 01: Init — Founder Mode

## Scenario
Single person initializes the SpecTeam workflow for the first time on a fresh repository.

## Prerequisites
- A Git repository with at least one commit
- No `.spec/` directory exists
- A design document directory exists (e.g., `./design/` with at least one `.md` file)

## Test Prompt
```
/spec-init
```

## Expected Interactions
1. AI asks for member code → reply: `alice`
2. AI asks for project goal (founder mode) → reply: `Build a unified API service for mobile and web clients`
3. AI asks for document directories → reply: `./design`

## Golden Dialogue Checkpoints

The live conversation should visibly include these checkpoints in order.

### Checkpoint 1 — Member code prompt
```text
**[SpecTeam init — Step 1]**

Please provide your member code (nickname / collaborator ID, e.g. alice, bob, dev-007).
This code identifies your documents across the shared workflow.
```

### Checkpoint 2 — Founder goal prompt
```text
**[SpecTeam init — Step 2: Set project goal]**

You are the first to initialize SpecTeam on this project. Please briefly describe the shared project goal / mission (1–3 sentences).
```

### Checkpoint 3 — Document directory prompt
```text
**[SpecTeam init — Step 3: Specify document directories]**

Please provide the local design document directories (comma-separated for multiple).
```

### Checkpoint 4 — Completion output
- Founder completion text says the project founder was established and documents were normalized into `.spec/design/alice/`
- Branch protection notice is printed after completion

## Verification Checklist

### Directory Structure
- [ ] `.spec/` directory created
- [ ] `.spec/design/alice/` contains copies of source documents
- [ ] Each copied file has `<!-- Spec Normalized Document -->` header

### Core Files
- [ ] `.spec/COLLABORATORS.md` exists with:
  - `**Main Branch**: {current branch name}`
  - Table row for `alice` with source dir, spec path, join date
- [ ] `.spec/THESIS.md` exists with North Star: "Build a unified API service..."
- [ ] `.spec/RULES.md` exists
- [ ] `.spec/SIGNALS.md` exists
- [ ] `.spec/INDEX.md` exists (auto-generated by parse)

### Git State
- [ ] `git config spec.member-code` returns `alice`
- [ ] `git config spec.main-branch` returns current branch name
- [ ] At least 2 commits created (init + parse)
- [ ] Commit messages start with `[SpecTeam]`

### Output
- [ ] Output includes branch protection notice (⚠️ [Branch Protection])
- [ ] Output follows the 6-section format (Execution Log / Current Identity / Diff-Aware Summary / Result Summary / Key Suggestions / Next Steps Recommended)
