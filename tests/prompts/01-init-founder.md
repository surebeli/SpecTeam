# Test 01: Init — Founder Mode

## Scenario
Single person initializes PhoenixTeam for the first time on a fresh repository.

## Prerequisites
- A Git repository with at least one commit
- No `.phoenix/` directory exists
- A design document directory exists (e.g., `./design/` with at least one `.md` file)

## Test Prompt
```
/phoenix-init
```

## Expected Interactions
1. AI asks for member code → reply: `alice`
2. AI asks for project goal (founder mode) → reply: `Build a unified API service for mobile and web clients`
3. AI asks for document directories → reply: `./design`

## Verification Checklist

### Directory Structure
- [ ] `.phoenix/` directory created
- [ ] `.phoenix/design/alice/` contains copies of source documents
- [ ] Each copied file has `<!-- Phoenix Normalized Document -->` header

### Core Files
- [ ] `.phoenix/COLLABORATORS.md` exists with:
  - `**Main Branch**: {current branch name}`
  - Table row for `alice` with source dir, phoenix path, join date
- [ ] `.phoenix/THESIS.md` exists with North Star: "Build a unified API service..."
- [ ] `.phoenix/RULES.md` exists
- [ ] `.phoenix/SIGNALS.md` exists
- [ ] `.phoenix/INDEX.md` exists (auto-generated by parse)

### Git State
- [ ] `git config phoenix.member-code` returns `alice`
- [ ] `git config phoenix.main-branch` returns current branch name
- [ ] At least 2 commits created (init + parse)
- [ ] Commit messages start with `[PhoenixTeam]`

### Output
- [ ] Output includes branch protection notice (⚠️ [Branch Protection])
- [ ] Output follows the 6-section format (执行日志 / 当前身份 / Diff感知摘要 / 结果摘要 / 关键建议 / 下一步推荐)
