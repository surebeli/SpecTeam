# Skill: parse (Core)

Scan all normalized documents, build the index, and detect changes.

**Write scope**: Only writes `INDEX.md` and `last-parse.json`. Never modifies THESIS.md, RULES.md, SIGNALS.md, DIVERGENCES.md, last-sync.json, or any file under `design/`.

## Parameters

None.

## Execution Steps (fixed rules)

1. Read `git config phoenix.member-code` to determine current identity. Apply identity guard.
2. Read `.phoenix/COLLABORATORS.md` to get all known collaborators.
3. Scan `.phoenix/design/` directory tree (organized by member code subdirectories).
4. Read `.phoenix/THESIS.md`, `.phoenix/SIGNALS.md`, `.phoenix/DIVERGENCES.md` (if exists).
5. Run `git log --oneline -1 -- .phoenix/THESIS.md` to get THESIS last-modified info.

### Step 6 — Generate INDEX.md

Write `.phoenix/INDEX.md` with the following structure:

```markdown
# PhoenixTeam INDEX

_Generated by parse @ {ISO timestamp}_

## North Star

> {THESIS.md 的 North Star 段落原文引用，不做 AI 改写或精简}
>
> _来源: THESIS.md (最后更新: {git log -1 THESIS.md 的 commit hash + date})_

## Decision Log (最近 3 条)

{从 THESIS.md 的 Decision Log 段提取最近 3 条，原文引用}

- 若无 Decision Log 段：_（暂无决策记录）_

## Divergences

{从 DIVERGENCES.md 提取状态摘要}

- 🔴 open: {count} | 🟡 proposed: {count} | ✅ resolved: {count}
{列出所有 open 和 proposed 条目：}
- D-{N}: {title} ({status}, {priority}) — {parties}

- 若无 DIVERGENCES.md：_（尚未执行 review）_
- 若无 open/proposed：_✅ 当前无待处理分歧_

## Signals

{从 SIGNALS.md 提取最新阻塞项和进度，原文引用}

## Document Tree

### {code-1}
- `design/{code-1}/file.md` — {one-line summary}

### {code-2}
- `design/{code-2}/file.md` — {one-line summary}

### shared (if exists)
- `design/shared/file.md` — {one-line summary}
```

**Rules for North Star section**:
- Must be a direct quote from THESIS.md, not an AI-generated summary or paraphrase.
- Include the git commit hash and date so readers know how fresh it is.
- If THESIS.md has been updated by align decisions, the quote reflects the latest version.

### Step 7 — Diff analysis

1. Run `git log --oneline -3 -- .phoenix/` and `git diff HEAD~1..HEAD -- .phoenix/` (if commits exist).
2. Generate **diff summary grouped by member code**.
3. If DIVERGENCES.md appears in the diff, highlight state transitions:
   - E.g., `"DIVERGENCES.md: D-001 open → proposed (alice 提议)"`

### Step 8 — Compare with last-parse.json and last-sync.json

1. Read `.phoenix/last-parse.json` (previous parse record).
2. Output **"变更内容"**:
   - Files added / modified / deleted since last parse
   - THESIS.md changes since last parse (quote the changed parts)
   - DIVERGENCES.md state changes since last parse
   - Key collaboration suggestions (based on real diff, must cite specific collaborator and diff content)
   - Example: `"发现 alice 的提案新增了 GraphQL 方案，与 THESIS 中 REST 优先策略冲突，建议触发 review"`

3. Read `.phoenix/last-sync.json` if it exists (source file tracking, written by update skill).
4. For the current user's source files, check if any hash differs from stored values:
   - If source drift detected, append a soft hint to output:
     ```
     💡 源文档漂移: {N} 个源文件自上次同步后已变更
        运行 /phoenix-update 将变更同步到工作区
     ```
   - This is informational only — parse never modifies source files or last-sync.json.

### Step 9 — Save and output

1. Save current parse state to `.phoenix/last-parse.json` (JSON with file list, timestamps, content hashes).
2. Output: INDEX.md preview + 【Diff 感知摘要】+ suggestions.
