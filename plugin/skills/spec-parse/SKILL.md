# Skill: parse (Core)

Scan all normalized documents, build the index, and detect changes.

**Write scope**: Only writes `INDEX.md` and `last-parse.json`. Never modifies THESIS.md, RULES.md, SIGNALS.md, DIVERGENCES.md, last-sync.json, or any file under `design/`.

## Parameters

None.

## Execution Steps (fixed rules)

1. Read `git config spec.member-code` to determine current identity. Apply identity guard.
2. Read `.spec/COLLABORATORS.md` to get all known collaborators.
3. Scan `.spec/design/` directory tree (organized by member code subdirectories).
4. Read `.spec/THESIS.md`, `.spec/SIGNALS.md`, `.spec/DIVERGENCES.md` (if exists).
5. Run `git log --oneline -1 -- .spec/THESIS.md` to get THESIS last-modified info.

### Step 6 — Generate INDEX.md (incremental when possible)

**Incremental update strategy** (reduces token cost when called as sub-skill):
- If `last-parse.json` exists and only specific collaborators have new commits → only regenerate the `## Document Tree` section for those collaborators. Preserve other sections (North Star, Divergences, Signals) if their source files are unchanged.
- If THESIS.md, DIVERGENCES.md, or SIGNALS.md appear in the diff since `last-parse.json` → regenerate their corresponding INDEX.md sections.
- If `last-parse.json` does not exist (first parse) → full generation of all sections.

Write `.spec/INDEX.md` with the following structure:

```markdown
# SpecTeam INDEX

_Generated by parse @ {ISO timestamp}_

## North Star

> {Direct quote of the North Star section from THESIS.md — no AI paraphrase or summarization}
>
> _Source: THESIS.md (last updated: {git log -1 THESIS.md commit hash + date})_

## Decision Log (last 3)

{Extract last 3 entries from THESIS.md Decision Log section, direct quote}

- If no Decision Log section: _(No decisions recorded yet)_

## Divergences

{Status summary extracted from DIVERGENCES.md}

- 🔴 open: {count} | 🟡 proposed: {count} | ✅ resolved: {count}
{List all open and proposed items:}
- D-{N}: {title} ({status}, {priority}) — {parties}

- If no DIVERGENCES.md: _(No review has been run yet)_
- If no open/proposed: _✅ No pending divergences_

## Signals

{Extract latest blockers and progress from SIGNALS.md, direct quote}

## Document Tree

### {code-1}
- `design/{code-1}/file.md` — {one-line summary}

### {code-2}
- `design/{code-2}/file.md` — {one-line summary}

### shared (if exists)
- `design/shared/file.md` — {one-line summary}
```

**Rules for North Star section**:
- Must be a direct quote from THESIS.md, not an AI-generated summary or paraphrase.
- Include the git commit hash and date so readers know how fresh it is.
- If THESIS.md has been updated by align decisions, the quote reflects the latest version.

### Step 7 — Diff analysis (anchor-based)

1. Run `git log --oneline -3 -- .spec/` for recent commit context.
2. Determine diff range using anchor from `last-parse.json`:

   | Condition | Diff command | Explanation |
   |-----------|-------------|-------------|
   | `last-parse.json` exists and contains a valid `head_commit` field | `git diff {head_commit}..HEAD -- .spec/` | Anchored: shows all changes since last parse |
   | `last-parse.json` missing or no `head_commit` | `git diff HEAD~1..HEAD -- .spec/` | Fallback: first parse or corrupted state |

3. Generate **diff summary grouped by member code**.
4. If DIVERGENCES.md appears in the diff, highlight state transitions:
   - E.g., `"DIVERGENCES.md: D-001 open → proposed (proposed by alice)"`

### Step 8 — Compare with last-parse.json and last-sync.json

1. Read `.spec/last-parse.json` (previous parse record).
2. Output **"Change Summary"**:
   - Files added / modified / deleted since last parse
   - THESIS.md changes since last parse (quote the changed parts)
   - DIVERGENCES.md state changes since last parse
   - Key workflow suggestions (based on real diff, must cite specific collaborator and diff content)
   - Example: `"Detected alice's proposal adds a GraphQL approach, conflicting with the REST-first strategy in THESIS; recommend triggering review"`

3. Read `.spec/last-sync.json` if it exists (source file tracking, written by update skill).
4. For the current user's source files, check if any hash differs from stored values:
   - If source drift detected, append a soft hint to output:
     ```
     💡 Source document drift: {N} source file(s) changed since last sync
        Run /spec-update to sync changes to the workspace
     ```
   - This is informational only — parse never modifies source files or last-sync.json.

### Step 9 — Save and output

1. Save current parse state to `.spec/last-parse.json` (JSON with file list, timestamps, content hashes, and `"head_commit": "<current HEAD hash>"` for anchored diff in next parse).
2. Output: INDEX.md preview + [Diff-Aware Summary] + suggestions.
